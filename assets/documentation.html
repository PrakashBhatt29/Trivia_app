About

Created by: Daniel McErlean

This trivia app utilizes the free Trivia Database API to give users a specified amount of questions in a variety of categories and difficulties. Built with Flutter for a school project.

Instructions / Page Uses

  • Home Page
    1. navigate to the Prompt Page
    2. navigate to the Highscore Page
    3. navigate to the Credits Page (here)
    4. quit the game
  • Prompt Page
    1. Enter username (required)
      • Entering a username already on the Highscore Page will add to that username's score
    2. Enter number of questions (required)
      • Max of 50
      • Anything other than a number will tell the user to enter only a number
      • Anything 0 or below, or above 50 will tell the user to enter a number between 1 and 50
    3. Choose Category
    4. Choose Difficulty
    5. If there are not enough questions in the chosen category and/or difficulty, the app will tell the user.
  • Game Page
    1. X amount of questions will be displayed, each with either 4 multiple choice answers, or 2 true/false answers
    2. The Appbar will display the category of each question
    3. The Appbar will change color depending on the difficulty of the question:
      • Green for Easy
      • Yellow for Medium
      • Red for Hard
    4. Users can select an answer, then press submit to lock-in that answer.
    5. After submitting, the app will:
      • change the color of the selected answer to red for incorrect
      • change the color of the selected answer to green for correct
      • display the correct answer
      • change the 'submit' button to the 'next' button, which will display the next question
    6. Pressing the 'next' button will bring the app back to the top of the screen if the next question is large enough for the app to scroll
    7. If the user presses the back button, a prompt to exit appears warning that progress will be lost when leaving in the middle of a game.
    8. After each question is answered, the app will:
      • Display a game over screen
      • Tell the user how many questions were answered correctly
      • Prompt the user to either play again, or go home
  • Highscore Page
    1. See each highscore during a session
    2. Currently, the highscore list will reset when the user exits the app.
  • data_provider
    1. Keeps track of the trivia API response
    2. Keeps track of the highscore list, and sorts it from highest to lowest score

Process

  1. Started out with an idea for a trivia game
  2. Created the pages and routing
  3. Set up the prompt page to receive api response
  4. Set up game page to display api response
  5. Set up data_provider to provide the users and api response to the different pages
  6. Styled app with dynamic sizes for different screens, and changed colors and font

Set Backs / Glitches

  • There is significant lag during the first round of every game, no matter how many questions. Every round after is rendered smoothly.
